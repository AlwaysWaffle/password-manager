# 🔐 Password Manager — Telegram Bot

> Менеджер паролей прямо в Telegram.  
> Пароли хранятся зашифрованными, мастер-пароль никогда не сохраняется на диск.

---

## ✨ Возможности

- 🔑 **Регистрация и вход** через мастер-пароль (8–32 символа) с подтверждением
- 🔒 **Шифрование паролей** алгоритмом Fernet (AES-128-CBC + HMAC-SHA256)
- 🧂 **Хеширование мастер-пароля** через SHA-256 + уникальная соль на каждого пользователя
- 🛡️ **Защита от брутфорса** — блокировка аккаунта после 5 неверных попыток на 10 минут
- ➕ **Добавление паролей** вручную или с генерацией
- 📋 **Просмотр сервисов** через inline-кнопки
- 🗑️ **Удаление записей** с подтверждением
- 🎲 **Генератор паролей** с выбором длины: 8 / 12 / 16 / 20 символов
- 🌐 **Встроенная поддержка прокси** — для пользователей из РФ, где Telegram заблокирован
- 💾 **SQLite** — локальная база, никаких внешних серверов

## 🚀 Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/AlwaysWaffle/password-manager.git
cd password-manager
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Настроить конфиг

Скопируй шаблон и заполни:

```bash
cp config.example.py config.py
```

Открой `config.py`:

```python
BOT_TOKEN = "ВАШ_ТОКЕН"      # токен от @BotFather

# Прокси — нужен для работы в России.
PROXY_TYPE = "ВАШ_ПРОТОКОЛ"       # протокол
PROXY_HOST = "ВАШ_IP_ПРОКСИ"      # адрес прокси
PROXY_PORT = "ВАШ_ПОРТ_ПРОКСИ"    # порт прокси
```

> Рекомендую использовать TG WS PROXY - https://github.com/Flowseal/tg-ws-proxy

> Если Telegram доступен напрямую — оставь поля пустыми и удали строку  
> `apihelper.proxy = ...` из `main.py`

### 4. Запустить бота

```bash
python main.py
```

## 🔐 Как работает шифрование

Безопасность строится на двух независимых механизмах.

**Мастер-пароль** никогда не хранится в открытом виде. При регистрации генерируется случайная соль (`secrets.token_hex`), и в базу записывается только `sha256(пароль + соль)`. При входе хеш пересчитывается и сравнивается — оригинал нигде не сохранён.

**Пароли сервисов** шифруются алгоритмом Fernet. Ключ шифрования каждый раз вычисляется из мастер-пароля (`sha256` → `base64`) и живёт только в памяти сессии. В базе хранится лишь зашифрованный токен — без мастер-пароля его невозможно расшифровать.

```
Мастер-пароль ──→ sha256(пароль + соль) ──→ хранится в таблице users
Мастер-пароль ──→ sha256 → base64 ──→ Fernet key (только в памяти)
Пароль сервиса ──→ Fernet(key).encrypt() ──→ хранится в таблице passwords
```

## ⚙️ Конфигурация

Все параметры — только в `config.py`:

| Параметр              | По умолчанию   | Описание                          |
| --------------------- | -------------- | --------------------------------- |
| `BOT_TOKEN`           | —              | Токен бота от @BotFather          |
| `PROXY_TYPE`          | —              | Протокол прокси                   |
| `PROXY_HOST`          | —              | IP-адрес прокси-сервера           |
| `PROXY_PORT`          | —              | Порт прокси-сервера               |
| `DB_PATH`             | `passwords.db` | Путь к файлу базы данных          |
| `MAX_FAILED_ATTEMPTS` | `5`            | Попыток входа до блокировки       |
| `LOCKOUT_MINUTES`     | `10`           | Минут блокировки после превышения |
| `TIME_FORMAT`         | `%H:%M:%S`     | Формат времени в логах            |

## 📸 Пример работы

```
/start
  └── новый пользователь → ввести мастер-пароль → подтвердить

Главное меню:
  ➕ Добавить пароль   📋 Мои сервисы
  🎲 Сгенерировать     🗑 Удалить пароль

Добавление пароля:
  → название сервиса → логин → пароль вручную / 🎲 сгенерировать (8/12/16/20)

Просмотр:
  → список сервисов → нажать → увидеть логин + пароль
```

## ⚠️ Важно

- `config.py` в `.gitignore` — **никогда не коммить токен в репозиторий**
- `passwords.db` в `.gitignore` — база остаётся только локально
- Если потеряешь мастер-пароль — восстановить сохранённые пароли **невозможно**
- После перезапуска бота нужно снова ввести `/start` и мастер-пароль (сессия в памяти)
